# 🚀 NexTalk Pro - نظام دردشة احترافي

<div dir="rtl">

## 📱 نظام دردشة متقدم مع لوحة تحكم عالمية

**NexTalk Pro** هو تطبيق دردشة حديث مبني على React و Firebase، مع لوحة تحكم إدارية متقدمة توفر تحكم كامل في المستخدمين والمحتوى.

</div>

---

## ✨ المميزات الرئيسية

<div dir="rtl">

### 🎯 مميزات الدردشة
- ✅ **دردشة فورية** - رسائل لحظية مع تحديث تلقائي
- ✅ **مصادقة آمنة** - تسجيل دخول برقم الهاتف عبر Firebase
- ✅ **دعم متعدد اللغات** - عربي، إنجليزي، فرنسي، إسباني، ألماني
- ✅ **حالة المستخدم** - متصل / بعيد / غير متصل
- ✅ **إشعارات فورية** - تنبيهات للرسائل الجديدة
- ✅ **صور ملف شخصي** - رفع وتعديل الصور الشخصية
- ✅ **بحث سريع** - البحث عن المستخدمين والرسائل

### 👑 لوحة التحكم الإدارية (Admin Dashboard)

#### 📊 نظرة عامة (Overview)
- **إحصائيات حية** - عدد المستخدمين النشطين والكلي
- **رسوم بيانية** - تحليل نشاط المستخدمين على مدار الأسبوع
- **معدلات النمو** - نسب الزيادة في عدد المستخدمين
- **إحصائيات الرسائل** - عدد الرسائل المرسلة اليوم والإجمالي
- **بطاقات تفاعلية** - عرض الإحصائيات بشكل جذاب ومتحرك
- **أكثر المستخدمين نشاطاً** - عرض Top 10 مستخدمين

#### 👥 إدارة المستخدمين (Users)
- **قائمة المستخدمين الكاملة** - عرض جميع المستخدمين المسجلين
- **فلترة متقدمة** - تصفية حسب الحالة (متصل/غير متصل/الكل)
- **بحث ذكي** - البحث بالاسم أو اسم المستخدم أو رقم الهاتف
- **معلومات مفصلة** - رقم الهاتف، تاريخ الانضمام، آخر ظهور
- **حذف المستخدمين** - إزالة أي مستخدم مع تأكيد
- **ترقية للمسؤولين** - تحويل أي مستخدم لمسؤول
- **منح الشارات** - إضافة شارات مميزة للمستخدمين (⭐💎🔥👑⚡)
- **إرسال إشعارات** - إرسال إشعار مباشر لأي مستخدم

#### 📢 البث الجماعي (Broadcast)
- **إرسال رسائل جماعية** - إرسال رسالة لجميع المستخدمين
- **إشعارات عامة** - إرسال إشعارات لجميع المستخدمين
- **إحصائيات البث** - عدد المستخدمين الذين سيستقبلون الرسالة
- **سجل البث** - عرض آخر 10 رسائل مرسلة
- **تنسيق غني** - دعم النصوص والإيموجي

#### 📈 التحليلات (Analytics)
- **رسوم بيانية للنشاط** - نشاط المستخدمين على مدار الأسبوع
- **نسب النمو** - تحليل نمو المستخدمين والرسائل
- **إحصائيات الحالة** - عدد المستخدمين المتصلين/غير المتصلين
- **مؤشرات التفاعل** - معدلات استخدام التطبيق
- **حلقات التقدم** - عرض النسب بشكل دائري جذاب

#### 🔐 الأمان (Security)
- **الجلسات النشطة** - عرض جميع الأجهزة المتصلة
- **إنهاء الجلسات** - فصل أي جهاز من الحساب
- **المصادقة الثنائية (2FA)** - إعداد Google Authenticator
- **رموز الاحتياط** - 8 رموز للطوارئ
- **سجل النشاط الأمني** - تتبع محاولات تسجيل الدخول
- **كشف الشذوذ** - تنبيهات للأنشطة المشبوهة
- **Device Fingerprinting** - تتبع الأجهزة المستخدمة

### 🎨 التصميم
- **واجهة حديثة** - تصميم Dark Mode احترافي
- **ألوان مريحة** - نظام ألوان متناسق ومريح للعين
- **رسوم متحركة** - حركات سلسة وجذابة
- **استجابة كاملة** - يعمل على جميع الشاشات (موبايل، تابلت، ديسكتوب)
- **تأثيرات بصرية** - Glow effects و Gradients احترافية

### 🔒 الأمان والحماية
- **تشفير البيانات** - جميع البيانات محمية
- **Rate Limiting** - حماية من الهجمات
- **TOTP 2FA** - مصادقة ثنائية قوية
- **Session Management** - إدارة آمنة للجلسات
- **XSS Protection** - حماية من هجمات XSS
- **CSRF Protection** - حماية من CSRF
- **Content Security Policy** - سياسات أمان صارمة

## 🚀 البدء السريع

<div dir="rtl">

### المتطلبات
- Node.js 18+ مثبت على جهازك
- حساب Firebase (مجاني)
- حساب GitHub (للنشر)
- حساب Vercel (مجاني)

### خطوة 1: إعداد Firebase

1. اذهب إلى [Firebase Console](https://console.firebase.google.com/)
2. أنشئ مشروع جديد
3. فعّل **Authentication** → Phone
4. فعّل **Realtime Database**
5. في الإعدادات → احصل على بيانات التكوين

### خطوة 2: إعداد المشروع محلياً

</div>

```bash
# 1. استنساخ المشروع
git clone <your-repo-url>
cd nextalk-pro

# 2. تثبيت المكتبات
npm install

# 3. إنشاء ملف البيئة
cp .env.example .env.local

# 4. تعديل .env.local بمعلومات Firebase الخاصة بك
# افتح الملف واستبدل القيم

# 5. تشغيل المشروع
npm run dev
```

<div dir="rtl">

### خطوة 3: النشر على Vercel

#### الطريقة الأولى: عبر GitHub (الأسهل)

1. ارفع المشروع على GitHub:
</div>

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <your-github-repo-url>
git push -u origin main
```

<div dir="rtl">

2. اذهب إلى [Vercel](https://vercel.com)
3. اضغط **New Project**
4. اختر المشروع من GitHub
5. أضف متغيرات البيئة من ملف `.env.local`:
   - `VITE_FIREBASE_API_KEY`
   - `VITE_FIREBASE_AUTH_DOMAIN`
   - `VITE_FIREBASE_PROJECT_ID`
   - `VITE_FIREBASE_STORAGE_BUCKET`
   - `VITE_FIREBASE_MESSAGING_SENDER_ID`
   - `VITE_FIREBASE_APP_ID`
   - `VITE_FIREBASE_MEASUREMENT_ID`
   - `VITE_FIREBASE_DATABASE_URL`
6. اضغط **Deploy**

#### الطريقة الثانية: عبر Vercel CLI

</div>

```bash
# 1. تثبيت Vercel CLI
npm i -g vercel

# 2. تسجيل الدخول
vercel login

# 3. النشر
vercel

# اتبع التعليمات وأضف متغيرات البيئة
```

## 📊 متطلبات Firebase

<div dir="rtl">

### Authentication
- Phone Authentication ✅ **مطلوب**
- Sign-in method: Phone

### Realtime Database
قواعد الأمان المقترحة:

</div>

```json
{
  "rules": {
    "users": {
      ".read": "auth != null",
      ".write": "auth != null",
      "$uid": {
        ".write": "$uid === auth.uid || root.child('users').child(auth.uid).child('isAdmin').val() === true"
      }
    },
    "messages": {
      ".read": "auth != null",
      ".write": "auth != null"
    },
    "broadcasts": {
      ".read": "auth != null",
      ".write": "root.child('users').child(auth.uid).child('isAdmin').val() === true"
    }
  }
}
```

## 🔐 الأمان

<div dir="rtl">

### ⚠️ مهم جداً

1. **لا ترفع ملف `.env.local` على GitHub**
2. **استخدم Firebase Rules لحماية البيانات**
3. **فعّل المصادقة الثنائية (2FA) للمسؤولين**
4. **راجع Authorized Domains في Firebase بانتظام**
5. **راقب سجل النشاط الأمني**

### مميزات الأمان المدمجة
- ✅ Rate Limiting (حماية من الهجمات)
- ✅ TOTP 2FA (مصادقة ثنائية)
- ✅ Session Management (إدارة الجلسات)
- ✅ Device Fingerprinting (تتبع الأجهزة)
- ✅ Anomaly Detection (كشف الأنشطة المشبوهة)
- ✅ XSS Protection (حماية من XSS)
- ✅ CSP Headers (Content Security Policy)
- ✅ Backup Codes (رموز احتياطية)
